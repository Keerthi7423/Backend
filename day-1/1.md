## 🧠 WHAT IS BACKEND?

Backend = logic + server + database

Example:
Frontend → UI (what user sees)
Backend → handles login, data, APIs

---

## ⚡ WHAT IS NODE.JS?

* JavaScript runtime
* Runs JS outside browser
* Used to build backend apps

Why Node.js?

* Fast
* Uses JavaScript everywhere
* Non-blocking (handles multiple tasks)

---

## 📁 PROJECT SETUP

1. Create folder
   mkdir day1-backend

2. Go inside folder
   cd day1-backend

3. Initialize project
   npm init -y

---

## ✍️ FIRST PROGRAM

Create file: index.js

console.log("Hello Backend 🚀");

Run:
node index.js

---

## 📦 MODULES (IMPORTANT)

Export:
module.exports = function

Import:
const add = require("./math");

---

## 📂 BUILT-IN MODULES

1. File System (fs)

Write file:
fs.writeFileSync("test.txt", "Hello");

Read file:
fs.readFileSync("test.txt", "utf-8");

2. OS Module

const os = require("os");
console.log(os.platform());

## 🎯 INTERVIEW QUESTIONS

1. What is Node.js?
   → A runtime to run JavaScript outside browser

2. What is backend?
   → Server-side logic handling data and requests

3. What is module.exports?
   → Used to export functions/variables from a file

4. What is require()?
   → Used to import modules

5. What is npm?
   → Node Package Manager for installing packages

6. What is package.json?
   → File that stores project details and dependencies

7. What is fs module?
   → Built-in module to handle file operations

8. What is non-blocking I/O?
   → Node handles multiple tasks without waiting

9. Difference between frontend and backend?
   → Frontend = UI, Backend = logic + server

10. What causes ENOENT error?
    → File not found or wrong path
