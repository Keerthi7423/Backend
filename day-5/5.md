DAY 5 — CLEAN ARCHITECTURE (ROUTES + CONTROLLERS)

---

## 🎯 GOAL

* Move from single file backend → structured backend
* Learn clean architecture
* Separate responsibilities
* Write scalable code like real companies

---

## 🧠 PROBLEM IN DAY 4

Everything was inside:

server.js

Problems:

❌ Hard to read
❌ Hard to maintain
❌ Hard to scale
❌ Not used in production apps

Example:

Routes + Logic + File operations all in one file

---

## SOLUTION (DAY 5)

Split backend into layers:

Client Request
↓

Routes
↓

Controllers
↓

Data/File/Database
↓

Response

This is called:

Separation of Concerns

---

## 📁 PROJECT STRUCTURE

day-5-products-api/
│
├── data/
│   └── products.json
│
├── controllers/
│   └── productController.js
│
├── routes/
│   └── productRoutes.js
│
├── server.js
└── package.json

---

1. data/products.json

---

Purpose:
Acts as temporary database

Code:

[]

Stores product data

Example:

[
{
"id": 1,
"name": "Laptop",
"price": 50000
}
]

---

2. controllers/productController.js

---

Purpose:
Contains business logic

Handles:

* Get all products
* Get product by id
* Add product
* Delete product

Example:

const getAllProducts = (req, res) => {
// logic here
}

Controller responsibilities:

* Process request
* Read data
* Write data
* Send response

---

3. routes/productRoutes.js

---

Purpose:
Defines API endpoints

Example:

router.get("/products", getAllProducts);

router.post("/products", addProduct);

router.delete("/products/:id", deleteProduct);

Route responsibility:
URL → controller mapping

---

4. server.js

---

Purpose:
Main entry point

Code:

const express = require("express");
const app = express();

const productRoutes = require("./routes/productRoutes");

app.use(express.json());

app.use("/", productRoutes);

app.listen(3000, () => {
console.log("Server running");
});

Everything becomes messy

With structure:

✅ Easy debugging
✅ Easy scalability
✅ Easy teamwork
✅ Professional architecture

## MODULARITY

Breaking code into reusable modules/files

Example:

Routes file
Controller file
Server file

Each file has one job

---

## SEPARATION OF CONCERNS

Routes → URL handling

Controllers → business logic

Data → storage

Server → app startup
