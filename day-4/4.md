## 🧠 WHAT IS EXPRESS.JS?

Express.js = Web framework for Node.js

Used to:

* Build APIs
* Handle routes easily
* Manage requests & responses

Why use Express?

* Less code
* Easy routing
* Middleware support

---

## ⚡ INSTALL EXPRESS

npm init -y
npm install express

---

## 🚀 BASIC SERVER

const express = require("express");
const app = express();

app.get("/", (req, res) => {
res.send("Hello Express 🚀");
});

app.listen(3000, () => {
console.log("Server running on port 3000");
});

---

## 🛣️ ROUTING

app.get("/", (req, res) => {
res.send("Home Page");
});

app.get("/about", (req, res) => {
res.send("About Page");
});

---

## 📡 JSON RESPONSE

app.get("/users", (req, res) => {
const users = [{ id: 1, name: "Keerthi" }];
res.json(users);
});

---

## 📥 REQUEST DATA

1. Query Params
   URL: /search?name=keerthi
   req.query.name

2. Route Params
   URL: /users/:id
   req.params.id

---

## 🧠 MIDDLEWARE

Middleware = function that runs before response

Example:

app.use((req, res, next) => {
console.log("Request received");
next();
});

next() → passes control to next function

---

## 📦 BODY PARSER

app.use(express.json());

Used to read JSON data from request body

---

## 🔄 HTTP METHODS

GET    → Fetch data
POST   → Send data
PUT    → Update data
DELETE → Remove data

---

## ⚠️ COMMON ERRORS

1. Cannot GET /
   → Route not defined

2. req.body undefined
   → Missing express.json()

3. Server not starting
   → Check node server.js

4. Port already in use
   → Change port number

## 🎯 INTERVIEW QUESTIONS & ANSWERS

1. What is Express.js?
   → A web framework for Node.js used to build APIs easily.

2. Why use Express?
   → Simplifies routing, middleware, and request handling.

3. What is middleware?
   → A function that runs between request and response.

4. What is req object?
   → Contains request data like params, query, body.

5. What is res object?
   → Used to send response to client.

6. What is res.json()?
   → Sends response in JSON format.

7. What is express.json()?
   → Middleware to parse JSON request body.

8. Difference between req.params and req.query?
   → params = route values (/users/:id)
   → query = URL values (?name=abc)

9. Difference between GET and POST?
   → GET = fetch data
   → POST = send data

10. What is routing?
    → Handling different URLs with different logic.
